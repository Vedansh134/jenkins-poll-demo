# How to use
- Clone the repository  
- Create a Jenkins job and configure it to poll the repository for changes
- Set the polling schedule (e.g., every 5 minutes)
- Save the job and let it run
- Check the Jenkins console output to see if the job is triggered when changes are made to the repository
- Make changes to the repository (e.g., add a new file or modify an existing file) and commit the changes
- Check the Jenkins console output again to see if the job is triggered by the changes you made
- You can also check the Jenkins build history to see the triggered builds and their status
- If the job is not triggered, check the Jenkins configuration and ensure that the polling schedule is set correctly and that the repository URL is correct
- If you encounter any issues, check the Jenkins logs for error messages and troubleshoot accordingly
- Remember to clean up the Jenkins job and repository after testing to avoid unnecessary clutter
